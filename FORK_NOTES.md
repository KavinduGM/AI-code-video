# AI-code-video — deterministic, voiceover-synced middle scenes

This is a **fork of the AI Video Creator app** for A/B testing one change: the
three middle scenes are built **entirely in code and synced to the voiceover**,
instead of being generated by Claude each render.

Everything else is identical to the original — same intros/outros (story-card
templates), captions, factory, music, transitions, safe-zone system.

## What changed

- **New:** [`src/main/pipeline/syncedscenes.ts`](src/main/pipeline/syncedscenes.ts) — the
  deterministic scene renderer.
- **Modified:** `src/main/pipeline/runner.ts` — middle scenes now try
  `buildSyncedSceneHtml` first; Claude generation stays only as an emergency
  fallback (e.g. an explainer with no quoted lines).

## How the synced scenes work

1. **Parse** the scene's `explainer` → scene type (plain **bands** / a text
   **box** / **marks + star**), heading, lines, and per-line marks (✓ / ✗ / ★).
2. **Draw** it in code — the box, the check/cross marks, and the filled star are
   code-drawn SVG/CSS, sized to fit the safe zone (same universal fit the static
   card uses). No AI in the loop.
3. **Sync** each line's fade-in to the moment its phrase is spoken, using the
   per-word timings ElevenLabs already returns (the same data that drives the
   karaoke captions). No extra API calls. Falls back to an even time-spread when
   a segment has no word timings.
4. Every reveal is a **one-pass** animation that ends fully visible
   (`fill-mode: both`, `iteration-count: 1`) → it physically cannot loop, flicker,
   or come out empty.

## How to compare

Render the **same script** in this fork and in the original app, and compare the
three middle scenes:

- **Original:** AI-designed scenes — varied look, but occasional loops / empty
  boxes / sparse frames, plus render-and-repair cost.
- **This fork:** the same 3-scene structure, drawn by code, text revealing in
  time with the narration — predictable, fast, no repair loop.

The intro and outro will look the same in both (they're already deterministic).
